Documenting the code you are building is a very important task for a software engineer, not only to make the code much more clear for your collaborators, but also for your clients. Xcode 13 offers a great framework for documentation purposes: **DocC**, which converts markdown-based text into rich documentation and displays it the Xcode documentation window or even in a website. Before getting started with the syntax and how to build the documentation we must make clear that DocC only builds documentation for packages or frameworks. That means that trying to build a documentation for a regular app is not allowed.
